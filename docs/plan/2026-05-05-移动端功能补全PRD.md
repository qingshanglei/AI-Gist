# AI-Gist 移动端功能补全 PRD

## 1. 背景与目标

AI-Gist 目前已经具备独立的移动端页面、Ionic 路由、提示词瀑布流、提示词增删改查、AI 配置、AI 生成器、云备份、导入导出、主题语言等基础能力。桌面端仍然拥有更完整的提示词管理能力，包括分类管理、标签筛选、表格/树形视图、批量操作、变量/Jinja 模板、快速优化、历史版本、生成历史等。

本阶段目标是在不破坏现有移动端架构的前提下，将桌面端大部分核心能力迁移到移动端，并用更适合手机的展示和交互方式承载。整体结构参考小红书的信息流展示方法：默认浏览、卡片预览、搜索优先、低学习成本、轻管理入口。

## 2. 产品原则

- 首页即内容：打开 App 后优先看到可浏览、可搜索、可直接使用的提示词卡片。
- 浏览像小红书，管理像工具：发现页降低学习成本，创作和我的页承载进阶管理。
- 常用操作前置：复制、收藏、填变量、AI 优化、编辑放在详情页底部或快捷菜单。
- 复杂能力渐进展示：变量、Jinja、历史版本、批量整理不占据新用户主路径。
- 不原样照搬桌面端：表格、树形、全局快捷键等桌面形态转换为移动端等价流程。

## 3. 信息架构

底部 Tab 建议调整为四个主入口：

- 发现：小红书式双列瀑布流，用于浏览、搜索、筛选、复制、收藏提示词。
- 创作：管理视图，用于新建、编辑、分类整理、批量处理、历史版本入口。
- AI：AI 生成、快速优化、生成历史、模型快捷选择。
- 我的：设置、数据备份、云同步、AI 配置、关于。

现有页面映射：

- `MobilePromptPage` 演进为发现页主体验。
- `MobilePromptDetailPage` 强化为阅读和使用页。
- `MobilePromptEditPage` 承载普通、变量、Jinja 三种编辑模式。
- `MobileAIGeneratorPage` 迁移进 AI Tab。
- `MobileAIConfigPage` 从底部主 Tab 下沉到我的页和 AI 页快捷入口。
- `MobileSettingsPage` 作为我的页的基础。

## 4. 功能范围

### P0 必须完成

- 发现页默认使用双列瀑布流，列表视图作为可选模式保留。
- 顶部搜索框常驻，搜索支持标题、描述、内容、标签。
- 横向频道栏支持全部、收藏、最近使用、分类、热门标签。
- 详情页底部固定操作栏：复制、填变量、AI 优化、收藏、更多。
- 普通提示词一键复制；变量/Jinja 模板进入填充后复制流程。
- 新建和编辑支持内容、标题、描述、分类、标签、图片、收藏。
- AI 生成使用首选模型，生成结果可保存为提示词。
- 设置中保留主题、语言、导入导出、云备份、AI 配置入口。

### P1 应该完成

- 分类管理：新增、重命名、删除、移动提示词到分类。
- 标签发现：热门标签、搜索标签、新建标签、标签详情列表。
- 快速优化：从详情页和编辑页进入，支持选择优化模板。
- 历史版本：编辑保存时创建历史版本，详情或编辑页可查看和回滚。
- AI 生成历史：可搜索、再次生成、保存为提示词、删除。
- AI 配置移动端完善：连接测试、请求测试、设置首选模型、系统提示词编辑。

### P2 可后续完成

- 批量整理：多选提示词后移动分类、批量删除、批量加标签。
- 最近使用和使用次数排行榜。
- 模板填充记录。
- 云备份冲突解决移动端流程优化。
- 移动端网络代理高级设置。

不迁移或不作为移动端主流程的桌面功能：

- 桌面表格视图和树形表格不原样迁移，只保留移动端等价管理流程。
- 桌面全局快捷键、窗口关闭行为、开机启动等 Electron 专属设置不进入移动端主流程。
- 开发环境实验室功能不进入正式移动端。

## 5. 页面流程

### 5.1 发现页

```text
[发现 Tab]
┌────────────────────┐
│ 搜索提示词 / 标签   │
│ 全部 收藏 最近 分类→ │
├─────────┬──────────┤
│ 卡片    │ 卡片      │
│ 图/标题 │ 标题/摘要 │
│ 标签 ❤️ │ 标签      │
├─────────┼──────────┤
│ 卡片    │ 卡片      │
└─────────┴────── ＋ ┘
```

交互逻辑：

- 默认进入发现页，加载瀑布流。
- 搜索框输入后即时刷新结果。
- 点击频道栏的分类或标签后，瀑布流按条件刷新。
- 点击卡片进入详情页。
- 长按卡片打开快捷菜单：复制、收藏、编辑、移动分类、删除。
- 右下角加号进入新建提示词。

### 5.2 提示词详情页

```text
卡片点击
→ [提示词详情]
┌────────────────────┐
│ 返回        ⋯       │
│ 标题                │
│ 描述 / 标签 / 分类   │
│ 图片区              │
│ 正文内容            │
│ 变量提示            │
├────────────────────┤
│ 复制  填变量  AI优化 │
└────────────────────┘
```

交互逻辑：

- 普通提示词点击复制后直接复制正文，并增加使用次数。
- 存在变量或 Jinja 标记时，复制按钮改为打开变量填充 Sheet。
- 填变量完成后生成最终提示词，支持复制或另存。
- AI 优化进入快速优化流程，结果可替换原文、保存副本或复制。
- 更多菜单包含编辑、历史版本、移动分类、删除、分享。

### 5.3 编辑页

```text
＋ / 编辑
→ [编辑提示词]
┌────────────────────┐
│ 取消          保存   │
│ 普通 | 变量 | Jinja │
│ 内容输入            │
│ 更多信息            │
│ 标题/描述/分类/标签  │
│ 图片                │
└────────────────────┘
```

交互逻辑：

- 默认展示内容输入，减少新建成本。
- 更多信息折叠区承载标题、描述、分类、标签、图片。
- 普通模式支持手动变量列表。
- Jinja 模式支持模板内容、变量识别、变量列表预览。
- 编辑已有提示词并保存时自动生成历史版本。
- 返回时如有未保存内容，弹出确认。

### 5.4 AI 页

```text
[AI Tab]
┌────────────────────┐
│ 生成提示词          │
│ 快速优化            │
│ 生成历史            │
│ 模型配置            │
└────────────────────┘
```

交互逻辑：

- 进入 AI 页时优先读取首选 AI 配置。
- 无配置时展示空状态，引导新增 AI 配置。
- 生成提示词需要填写需求描述，默认自动保存，也可关闭自动保存后手动保存。
- 快速优化支持从 AI 页独立进入，也支持从详情页携带当前提示词进入。
- 生成历史按时间倒序展示，支持再次生成、保存为提示词、删除。

### 5.5 我的页

```text
[我的 Tab]
┌────────────────────┐
│ 数据与同步          │
│ 云备份              │
│ AI 配置             │
│ 外观与语言          │
│ 关于                │
└────────────────────┘
```

交互逻辑：

- 数据导入导出继续使用移动端文件和分享能力。
- 云备份维持 WebDAV 和 iCloud 配置能力。
- AI 配置放在我的页，同时 AI 页提供快捷入口。
- 外观和语言保持移动端原生选择器体验。

## 6. 用户故事

- 作为新用户，我希望打开 App 后直接看到可浏览的提示词卡片，这样不需要先理解分类和表格。
- 作为重度用户，我希望可以通过搜索、分类、标签、收藏快速缩小范围，这样能在手机上快速找到常用提示词。
- 作为提示词使用者，我希望详情页一键复制，遇到变量模板时先让我填变量再复制。
- 作为内容维护者，我希望在手机上新建、编辑、加图、加标签、移动分类，而不依赖桌面端。
- 作为 AI 用户，我希望在任意提示词详情里直接调用快速优化，并选择替换、另存或复制结果。
- 作为配置用户，我希望移动端能管理 AI 配置、测试连接、设置首选模型。
- 作为谨慎用户，我希望编辑保存后能查看历史版本并回滚。
- 作为多设备用户，我希望移动端能导入导出和使用云备份恢复数据。

## 7. 关键状态与边界

- 无提示词：发现页展示空状态，引导新建或 AI 生成。
- 无 AI 配置：AI 页和 AI 优化入口展示配置引导。
- 搜索无结果：保留搜索词，提供清空筛选入口。
- 变量未填完整：提示必填变量，禁止复制最终内容。
- AI 生成失败：保留用户输入，展示错误原因，允许重试。
- 导入数据成功：提示后刷新本地数据。
- 删除分类：需要确认处理分类下提示词，默认将提示词移动到未分类。
- 删除提示词：二次确认，删除后返回上一级或刷新列表。

## 8. 验收标准

- 用户首次进入移动端时，默认看到瀑布流内容，不需要进入筛选弹窗才能浏览。
- 搜索、分类、标签、收藏筛选可以组合使用，并且结果数量和空状态清晰。
- 普通提示词可以从卡片或详情页快速复制。
- 变量或 Jinja 提示词不会直接复制未填充模板，而是引导填充。
- 新建和编辑流程在手机屏幕上不出现控件重叠和文字溢出。
- AI 生成和快速优化在无配置、有配置、请求失败三种状态下都有明确反馈。
- 历史版本可以查看并回滚，回滚后当前提示词内容更新。
- 导入导出和云备份流程在 iOS 和 Android 上有明确可用性提示。

## 9. 测试计划

- 发现页：瀑布流、列表切换、搜索、分类、标签、收藏、无限滚动、滚动位置恢复。
- 详情页：复制普通提示词、填充变量后复制、图片预览、收藏、删除、编辑返回刷新。
- 编辑页：普通、变量、Jinja 模式保存，标签新增，图片添加删除，未保存返回确认，历史版本生成。
- AI：无配置空状态、有首选模型默认选择、生成保存、快速优化替换/另存/复制、历史记录管理。
- 设置：导入导出、云备份新增/恢复、主题语言切换、AI 配置测试和首选设置。
- 适配：iPhone SE、常规 iPhone、Android 中屏；浅色和深色；离线和接口失败提示。

## 10. 默认假设

- 日期使用当前环境日期：2026-05-05。
- 移动端优先迁移能在手机上自然完成的核心功能。
- 小红书参考仅用于信息流、卡片、频道、底部 Sheet 和低学习成本交互，不复制社交关系能力。
- 现有 IndexedDB、API 服务和移动端 Ionic 技术栈继续复用，优先调整页面组织和交互。
- 底层数据模型非必要不改动，除非变量、历史版本或快速优化在实现时发现移动端缺失必要字段。
